# DuckFlow Manager Agent

Compatibility shim for agent systems that read a single root-level prompt file.

The canonical directory-style manager prompt lives at:

- `.agents/duckflow-manager.md`

Use that file when the environment supports `.agents/`.

Use very short prompts such as:

- `Use .agents/duckflow-manager.md and start a new project.`
- `Use .agents/duckflow-manager.md and refresh DuckFlow from the current repo state.`

Do not assume fixed doc, SQL, or OpenAPI paths. The manager is expected to discover existing files by pattern and scaffold minimal starters when key project-facing inputs do not exist yet.
