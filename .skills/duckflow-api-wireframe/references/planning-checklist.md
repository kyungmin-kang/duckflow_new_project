# API and Wireframe Planning Checklist

## Targets

- correct mode
- correct base URL
- reusable naming

## Tests

- one request per test
- method/path/contract set
- linked nodes/pages when useful

## Suites

- group by page flow, smoke check, or contract area
- avoid giant kitchen-sink suites

## Page plans

- route path set
- wireframe node linked
- components placed
- load behavior assigned
- linked tests attached

## Sync checks

- API nodes in the map still match saved tests
- contract nodes still match validation expectations
- page plans still match current UI/component intent
