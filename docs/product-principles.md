# Product Principles

1. The map should explain the product architecture, not just store disconnected notes.
2. API tests should stay linked to UI components and page plans whenever possible.
3. External data ingestion should be represented through metadata manifests, not by putting secrets into DuckFlow.
4. Docs, YAML, and in-app state should round-trip cleanly.
5. Human operators should mainly review and steer; coding agents should do the setup and updates.
