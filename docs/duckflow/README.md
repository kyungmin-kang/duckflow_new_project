# DuckFlow Project Kit

This folder holds the DuckFlow-facing assets that belong in a project repo.

Use it for:

- project-local YAML bundles
- operator notes and onboarding docs
- extractor handoff notes
- any project-specific DuckFlow conventions that should live with the codebase

Recommended files:

- `project-template.yaml`
- `examples/`
- `operator-notes.md`

Keep app runtime code in the standalone DuckFlow app repo. Do not copy the whole app into each project.
