# DuckFlow Operator Notes

Fill this with project-specific guidance for coding agents and human operators:

- which docs are the most authoritative
- where SQL, OpenAPI, and sample data live
- which local ports are expected during development
- which DuckFlow YAML bundles are maintained manually vs generated
- what extraction hooks exist under `tools/duckflow/`
- which sensitive domains should never be previewed through DuckFlow

Use this file when the repo does not match the default starter layout.

Examples:

- docs live in `docs/planning/` instead of `docs/`
- SQL is split across `sql/schema/`
- OpenAPI is split across `contracts/`
- the local app runs on port `4000` instead of `3001`
