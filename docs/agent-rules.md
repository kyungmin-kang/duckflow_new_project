# Agent Rules

- Use `.agents/duckflow-manager.md` as the single DuckFlow entrypoint unless the task is obviously narrow.
- Keep map, execution control, API tests, and page wireframe plans coherent together.
- Prefer updating project docs and YAML alongside in-app state changes.
- Do not introduce new DuckFlow app features from this project repo.
- Keep Cloudflare and SSH concerns out of this starter unless explicitly needed later.
